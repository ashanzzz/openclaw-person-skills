## 14 条管理原则（工厂应用版）

### 原则 2-4：建立连续流（Flow）

**把工序连成一条连续的线**：
- 停滞=问题（库存、等待、返工都是信号）
- 单元式生产（Cell）替代大批量流水线
- 均匀化生产（Heijunka）：用生产节拍（TAKT Time）决定每个工序节奏

```
节拍时间 = 有效工作时间 / 客户需求量
例：一天工作8小时=28800秒，客户需求2000件
节拍时间 = 28800/2000 = 14.4秒/件
```

### 原则 5：停机和立即解决

**机器出问题 → 立即停机 → 立刻分析 → 解决后再开**：
- 不让机器带病运转（自动化/Jidoka精神）
- 5个为什么 + 现场观察（Gemba Walk）
- 问题出现时第一时间感知，不是积累到最后爆发

### 原则 6：标准化是改善的基础

每道工序都要有书面作业标准：
```
1. 标准节拍时间
2. 标准在制品数量（WIP）
3. 标准作业顺序
4. 标准质量检验点
```
**没有标准化，就没有改进的依据。**

### 原则 7：目视化管理（Andon）

让任何人都能立即看出异常状态：
- 异常灯/看板颜色/停线绳
- 信息透明化 → 责任明确化
- 异常必须可视化，否则会被忽略

### 原则 9：现场现物现地（Genchi Genbutsu）

**领导必须到现场观察，而不是看报表**：
- 每天至少一次走到最可能有问题的工序
- 亲眼看操作员的实际操作
- 问操作员遇到了什么问题

```
Walk 顺序：
1. 从后工序往前工序走（后工序是客户）
2. 看看板：WIP是否超标？有没有异常信号？
3. 看设备：有没有异常振动/噪音/气味？
4. 看操作员：有没有做标准外的动作？
5. 问问题：你觉得哪里最卡？为什么？
```

### 原则 12：供应商是合作伙伴

- 不是"压价"，而是"共同改善"
- 定期到供应商现场一起改善
- 帮助供应商提升能力，双方都受益

| 工具 | 用途 |
|------|------|
| **看板（Kanban）** | 拉式生产信号系统 |

## 改善三阶段路线图

```
阶段一（第1-3月）：现场基础
1. 推行5S（整理/整顿/清扫）
2. 建立目视化管理（异常灯/看板）
3. 确定每道工序的作业标准

阶段二（第4-6月）：建立连续流
4. 按产品族重新布局工序（Cell化）
5. 消除明显搬运和等待
6. 建立单元生产

阶段三（第7-12月）：拉式系统
7. 导入看板
8. 均匀化排产（Heijunka）
9. 建立供应商拉式补料
```

---

## 5S 实施标准

```
S1 整理：区分要/不要，清除不要的物品
S2 整顿：每件物品有固定位置，标识清晰
S3 清扫：设备地面无灰尘/铁屑/油污
S4 清洁：建立清扫/检查标准，形成制度
S5 素养：所有人不需提醒自觉执行
```

## 看板使用标准

```
启动条件：
- 工序间物流稳定
- 质量问题已控制在可接受范围
- 员工已掌握标准化作业

最小库存 = 平均需求量 × 等待补充时间
例：每天用200件，补充需要1天 → 看板最小数量=200件
超过最大库存：停止生产
出现质量问题：拉动异常看板，整条线停线检查
```

## 丰田之道 vs TOC（《目标》）

| 丰田之道 | TOC |
|---------|-----|
| 拉式生产（Pull） | DBR 的 Rope |
| 均匀化（Heijunka） | 瓶颈设定节奏（Drum） |
| 停机立即解决 | 保护瓶颈不让坏件通过 |
| 标准化是改善基础 | 标准化让改善可见 |
| 目视化暴露问题 | 瓶颈问题立即可见 |
