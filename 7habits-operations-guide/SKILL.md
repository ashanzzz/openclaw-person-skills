# SKILL.md — 7Habits 运营指南

> 基于《高效能人士的七个习惯》（史蒂芬·柯维）  
> 用途：回答工厂运营、公司运营、个人运营问题  
> 核心原则：产出/产能平衡（P/PC Balance）

---

## 七个习惯框架

```
独立（个人领域）
├── 习惯一：积极主动
├── 习惯二：以终为始
└── 习惯三：要事第一

互赖（公众领域）
├── 习惯四：知彼解己（双赢思维）
├── 习惯五：知彼解己（先理解别人，再求被理解）
└── 习惯六：统合综效（协同增效）

持续更新
└── 习惯七：不断更新
```

---

## 核心原则速查

### P/PC Balance（产出/产能平衡）

> 效能的基础是产出与产能的平衡

- **P（Production）**：产出——直接结果（产品、收入、客户）
- **PC（Production Capability）**：产能——产生产出的能力（设备、人员、系统、关系）

| 场景 | P（产出） | PC（产能） |
|------|---------|-----------|
| 工厂 | 订单交付 | 设备保养、员工培训 |
| 销售 | 签单 | 客户关系维护 |
| 管理 | 业绩 | 团队能力成长 |
| 个人 | 工作成果 | 身体/精神/智力/社交 |

**原则：杀鸡取卵透支产能，只顾眼前产出，最终两败俱伤。**

## 习惯一：积极主动（Be Proactive）

**定义**：采取主动，为自己的行为负责；依据原则而非情绪/环境做决定。

### 工厂/公司场景

| 低效能（被动） | 高效能（主动） |
|---------------|---------------|
| 设备坏了才修 | 预防性维护计划 |
| 客户投诉才处理 | 主动客户关系维护 |
| 员工离职才招人 | 定期沟通、留存机制 |
| 问题发生了再想办法 | 风险预判，提前布局 |

### 回答要点
- 遇到困境时，先问："**我现在能控制什么？**"
- 把注意力从"不可控因素"（竞争对手、市场、政策）转向"可控因素"（自身效率、团队能力、产品品质）
- 用"我能/我会/我选择"代替"但愿/不得不/没办法"

---

## 习惯二：以终为始（Begin with the End in Mind）

**定义**：先有明确愿景/目标，再行动；所有决定以原则为中心。

### 工厂/公司场景

关键问题：**你的工厂/公司3年后要成为什么样子？**

| 问题维度 | 示例 |
|---------|------|
| 客户定位 | 服务哪类客户？追求量还是质？ |
| 产品定位 | 哪些品类？核心技术是什么？ |
| 规模定位 | 年产值目标？人员规模？ |
| 能力定位 | 核心竞争力是什么？ |

### 使命宣言框架

```
[你的名字/公司]的使命是：
帮助（谁）通过（什么方式）实现（什么结果）

例：帮助制造企业通过精益管理实现高效交付
```

### 回答要点
- 任何决策前，先问："**这是否符合我们的长期愿景？**"
- 公司层面：使命→愿景→价值观→战略→年度目标
- 个人层面：人生角色→3年目标→年度目标

---

## 习惯三：要事第一（Put First Things First）

**定义**：做重要的事，而不是紧急的事。

### 时间管理矩阵

| 象限 | 内容 | 行动 |
|------|------|------|
| **Q1** | 紧急又重要 | 立即做（危机、紧急问题） |
| **Q2** | 重要不紧急 | **计划做**（规划、预防、关系建设） |
| Q3 | 紧急不重要 | 尽量委托他人 |
| Q4 | 不紧急不重要 | 删除 |

### 工厂/公司运营重点

**Q1优先处理**：设备重大故障、大客户投诉、质量事故

**Q2重点建设**（决定长期竞争力）：
- 设备预防性维护计划
- 员工技能培训体系
- 供应商关系管理
- 核心客户深耕
- 工艺标准化/SOP建设

### 回答要点
- 问："**这件事是Q几？**"帮助区分优先级
- 原则：每天留出"不被干扰的Q2时间"处理重要不紧急的事

## 习惯四：双赢思维（Think Win-Win）

**定义**：所有合作都以"我赢你也赢"为基础。

### 工厂场景的应用

| 合作方 | 低效能思维 | 双赢思维 |
|--------|-----------|---------|
| 客户 | 压价、延期付款 | 品质保障→长期信任→合理利润 |
| 供应商 | 无限压价 | 合理利润→稳定供货→品质保障 |
| 员工 | 压低工资 | 合理报酬→稳定团队→品质稳定 |
| 外协伙伴 | 压工价 | 合理工价→稳定交期→质量可靠 |

### 回答要点
- 冲突时问："**有没有让双方都赢的方案？**"
- 宁可放弃单次交易，不破坏长期合作关系
- 双赢协议：明确期望→明确标准→明确时限

## 习惯五：知彼解己（Seek First to Understand, Then to Be Understood）

**定义**：先真正理解对方，再寻求被理解。

### 沟通层次

```
第1层：听（听表面说了什么）
第2层：复述（确认理解对方意思）
第3层：共情（理解对方情绪和立场）
第4层：无声理解（全然接纳对方的感受）
```

### 工厂/公司场景

| 场景 | 低效能做法 | 有效做法 |
|------|-----------|---------|
| 老板布置任务 | 不理解就埋头做 | 先复述确认→再行动 |
| 员工汇报问题 | 打断给建议 | 先听完整→问开放式问题→再回应 |
| 客户投诉 | 辩解/反驳 | 先共情理解→再解释方案 |
| 跨部门协作 | 各说各话 | 先了解对方立场→再表达己方需求 |

### 回答要点
- 沟通问题先诊断："**对方需要的是被理解，还是解决方案？**"
- 给出建议前，先问对方想法，避免"过早给出答案"

## 习惯六：统合综效（Synergize）

**定义**：整体大于部分之和；差异是优势，不是弱点。

### 核心思维

```
1 + 1 = 3 或更多
两个人的不同观点 → 第三选择 → 比各自方案更好
```

### 工厂/公司场景

| 场景 | 低效能 | 统合综效 |
|------|--------|---------|
| 技术问题 | 各部门独立解决 | 跨部门协同找最优解 |
| 与外协合作 | 压价/服从 | 发挥各自优势→共同提升 |
| 团队冲突 | 压制/妥协 | 差异碰撞→更好的第三方案 |
| 新业务探索 | 单方面决策 | 集思广益→共创新可能 |

### 回答要点
- 遇到僵局问："**能否把双方最好的想法合并，创造第三方案？**"
- 差异 = 信息源多元化，是创新的土壤

## 习惯七：不断更新（Sharpen the Saw）

**定义**：在身体、精神、智力、社会/情感四个层面持续投资自己。

### 四个维度

| 维度 | 内容 | 工厂/公司类比 |
|------|------|-------------|
| 身体 | 运动、睡眠、饮食 | 设备保养、工厂维护 |
| 精神 | 价值观、愿景、反思 | 企业文化、使命坚守 |
| 智力 | 阅读、学习、写作 | 技能培训、流程优化 |
| 社会/情感 | 人际关系、给予价值 | 客户关系、团队建设 |

### 工厂场景：设备与系统的"不断更新"

- **预防性维护 > 事后维修**（身体维度）
- **工艺标准化 + 持续改进**（智力维度）
- **员工培训体系**（社会维度）
- **团队文化**（精神维度）

### 回答要点
- 问："**你上一次在哪些方面投入了产能？**"
- 提醒：忙碌≠高效，投资产能才能持续产出

---

## 回答问题框架

当用户问运营问题（工厂/公司/个人）时：

1. **诊断问题属于哪个习惯的范畴**
2. **引用P/PC Balance原则**分析问题
3. **用对应习惯的原则给出建议**
4. **结合实际场景给出可操作的建议**

---

## 使用方式

### 触发条件
- 用户问工厂运营、公司经营、个人成长/效率相关问题
- 用户感到迷茫、焦虑、困惑
- 用户做重大决策前

### 参考优先顺序
1. 先理解问题的本质（习惯五）
2. 判断问题紧迫性（习惯三矩阵）
3. 给出对应习惯的建议（习惯一/二/四/六/七）
4. 始终回归P/PC平衡原则
