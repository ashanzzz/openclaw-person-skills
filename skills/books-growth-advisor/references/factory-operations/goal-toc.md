# 工厂运营：目标与约束理论（TOC）

## 核心问题：为什么工厂效率低、产出慢？

**答案**：整个系统的产出速度由最慢的环节（瓶颈）决定。优化非瓶颈只是浪费。

## 三大量化指标

| 指标 | 定义 | 工厂例子 |
|------|------|---------|
| **Throughput（产出）** | 通过销售收回金钱的速度 | 每日发货量 × 单价 |
| **Inventory（库存）** | 投入购买待售品的全部资金 | 原料 + 在制品 + 成品 |
| **Operating Expense** | 将库存转化为产出所花的钱 | 人工 + 水电 + 折旧 |

> **目标**：同时 ↑ 产出、↓ 库存、↓ 运营费用

## TOC 五步法（完整版）

### STEP 1：识别瓶颈

**问题信号**（凭观察不凭猜测）：
- 某台机器/工序前堆积大量半成品
- 某台设备停机，整个车间就得等
- 整体产出由某个特定环节决定

**诊断方法**：走完整个生产流程，记录每个工序的产出速度和等待时间。找到1-3个瓶颈（不会很多）。

### STEP 2：挖尽瓶颈（不花钱）

- 瓶颈不停工（换型/清洁安排在计划停机时间）
- 瓶颈前100%检验，不让坏件浪费瓶颈时间
- 瓶颈前设置缓冲库存（0.5-1天量），保护瓶颈不被上游波动打断
- 瓶颈的操作员不安排任何非生产任务

### STEP 3：一切服从瓶颈

- 非瓶颈不要比瓶颈跑更快（跑快了只是堆积）
- 瓶颈设定整个系统的节奏（Drum）
- 上游按瓶颈需求供料（拉式补充）

### STEP 4：提升瓶颈（需要投资时）

- 购入或租赁新设备
- 加班/增加班次
- 外包瓶颈工序
- 雇佣更多人

### STEP 5：重复循环

打破一个瓶颈后，回到STEP 1找新瓶颈。

## DBR 系统（Drum-Buffer-Rope）

TOC 在制造中的具体实现：

| 组件 | 含义 | 工厂实例 |
|------|------|---------|
| **Drum（鼓）** | 瓶颈设定整个系统的节奏 | CNC每天产出200件，整条线以它为准 |
| **Buffer（缓冲）** | 保护瓶颈的库存缓冲 | CNC前备2天库存，确保不饿 |
| **Rope（绳子）** | 拉式信号，上游按需供料 | 仓库看缓冲低了，通知上一工序补料 |

## 工厂常见误区

| 误区 | 后果 |
|------|------|
| 每台机器高效运转 | 局部最优，整体更慢 |
| 库存多=产能足 | 掩盖问题，资金积压 |
| 补人=提高产出 | 瓶颈不解决，加人只是更多堆积 |
| 用局部指标考核 | 各部门只顾自己，整体失调 |

## SOP：诊断"为什么工厂不赚钱"

```
1. 确认目标：赚钱（不是"多产"、不是"填满产能"）
2. 收集数据：
   - 各工序产出速度（节拍时间）
   - 各工序等待时间
   - 在制品库存量（WIP）
   - 订单延误情况
3. 找到瓶颈（通常1-3个）
4. 按五步法处理
```

## 瓶颈识别 Checklist

```
[ ] 哪个工序前堆积最多在制品？
[ ] 哪台机器/工序最常导致停工等待？
[ ] 整体产出由哪个环节决定？
[ ] 瓶颈前缓冲是否足够保护瓶颈？
```

## 保护瓶颈的正确做法

```
1. 在瓶颈前设置质量门（100%检验，不让坏件通过）
2. 瓶颈前设置缓冲库存（0.5-1天）
3. 瓶颈不参与换型/清洁等非生产时间
4. 瓶颈操作员不安排非生产任务
5. 瓶颈的维护放在计划停机时间
```

## 非瓶颈的正确做法

```
1. 不要让机器100%运转（留余量吸收波动）
2. 按瓶颈节奏供料，不是按自己最大速度生产
3. 优先清理瓶颈前的等待队列
4. 换型时间优化非瓶颈工序（减少准备时间）
```
