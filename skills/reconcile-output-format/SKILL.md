# 财务核对结果输出格式

## 标准列定义

### ERPNext 发票明细（输出格式）

每行必须包含以下列：

| 列名 | 说明 | 数据来源 |
|------|------|---------|
| 凭证号 | ERPNext 系统内部编号 | `name` |
| 发票号码 | 税务局认证编号 | `bill_no` |
| 日期 | 凭证日期 | `posting_date`（取月日部分如 01-27） |
| 供应商 | 供应商名称 | `supplier_name` |
| 物料名称 | 行项目物料名 | `items[].item_name` |
| 规格 | 物料规格型号 | `items[].custom_guige_xinghao` |
| 备注 | 费用用途说明 | `items[].custom_备注` |
| 数量 | 行数量 | `items[].qty` |
| 金额 | 行金额（不含税） | `items[].amount` |
| **匹配状态** | 勾稽结果 | 动态标注 |

### 匹配状态标注规则

| 状态 | 标注 | 说明 |
|------|------|------|
| 已匹配 | ✅ 已匹配 `{财务凭证号}` | 发票已找到对应银行凭证 |
| 未匹配 | ❌ 未匹配 | 发票在银行无对应出账，需追查 |
| 无需匹配 | ⏭ 无需匹配 | 日常费用/工资/税费/理财等，正常不录入ERPNext |

## 带标记账单（Bank Statement）格式

> 当用户要求生成「带标记账单」时，使用此格式。
> 结构为标准银行账单（日期/摘要/借方/贷方/余额），备注列只写 **ERPNext风格实际备注**（原始备注或合理补充）。
> 备注里不写“已匹配/未匹配/无需匹配”等状态字样。

### 列定义

| 列 | 说明 |
|----|------|
| 日期 | YYYY/MM/DD，如 2026/01/06 |
| 摘要 | 财务账单原始摘要，无前缀* |
| 借方 | 收入金额，无则填 0 |
| 贷方 | 支出金额，无则填 0 |
| 余额 | 累计余额 |
| 备注 | ERPNext原始备注（如有）/ 我们补充的实际业务备注（如知道）；不知道如何写的备注可以留空 |

### 颜色规则

| 颜色 | 含义 |
|------|------|
| 绿色 | ✅ 已匹配 |
| 红色 | ❌ 未匹配（需追查） |
| 黄色 | ⏭ 无需匹配（日常费用/工资/税费/理财） |
| 蓝色 | 收入（货款/补贴） |

## 输出前硬校验（每次必做）

在回复用户前，必须自检：

- 第一部分是否包含完整 10 列：
  `凭证号 | 发票号码 | 日期 | 供应商 | 物料名称 | 规格 | 备注 | 数量 | 金额 | 匹配状态`
- 如果用户要看“发票明细”，不得只给简版汇总（比如只给凭证号+金额）。
- 对话过长时可“分段发”，但每段都要保留表头，不得删列。
- 若字段缺失，视为输出不合格，先补齐再发。

## 关键规则

1. **一行一个物料明细**，不是按发票汇总
2. 匹配状态列：直接写 `✅ 已匹配 {财务凭证号}` 或 `❌ 未匹配`
3. 多行共用同一财务凭证时，每行都标注同一个凭证号
4. 金额精确到小数点后两位，显示千分位分隔符
5. 未匹配行放在已匹配行之后
6. 分类标题根据当前核对类型动态变化（电汇/现金/应付）
7. 带标记账单备注列：参照 `finance-remark-mapping` skill，写 **ERPNext风格实际备注**（例如：`2025年12月电费（1月支付上月电费）`、`电汇手续费`）。
8. 备注填充优先级（硬规则）：
   - 先用 ERPNext 明细表已有备注；
   - ERPNext 无备注时，再按业务语义自然补写；
   - 仍不确定时留空。
9. 备注列禁止写匹配状态词（如：已匹配/未匹配/无需匹配）；这些状态放在颜色或单独汇总里。
10. **拆分精细化原则（通用规则）**：ERPNext 已匹配发票拆分越细越好：主物料/副物料/手续费分别成行。
11. 进度文件路径：`learning/finance-YYYY-MM/<分类>核对_working.md`
12. **禁止简化输出字段**：即使在对话框里展示，也必须保留完整列；如果内容过长，分批发送，不能删列。
