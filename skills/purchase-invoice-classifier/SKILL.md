# 采购发票四级分类技能

## 分类标准（硬规则，优先级从高到低）

### 第一刀：按发票号码格式（bill_no）

| bill_no 样式 | 判断 | 说明 |
|---|---|---|
| `0` | ⚠️ **无发票** | 系统未录入真实发票号 |
| `260000000` | ⚠️ **无发票** | 虚票号，非真实税务发票 |
| 其他数字（18位等） | ✅ 有发票 | 真实发票号码，继续第二步判断 |

### 第二刀：按付款方式（Payment Entry 推算，有电汇优先电汇）

| 付款方式 | 分类标签 |
|---|---|
| **电汇** | 🏦 **电汇** |
| **现金** | 💵 **现金** |
| 未找到对应 Payment Entry 且 outstanding=0 | ✅ 已确认已付（备用分类） |
| 未找到对应 Payment Entry 且 outstanding>0 | 🔴 **未付款** |

> **电汇优先规则**：同一张发票可能同时存在现金 PE 和电汇 PE（例如电汇跨月到账），分类时电汇 > 现金，以最终实际付款方式为准。

### 最终四级分类

| 分类 | 判断条件 | emoji |
|---|---|---|
| 🏦 电汇 | bill_no=真实号码 **且** payment_method=电汇 | |
| 💵 现金 | bill_no=真实号码 **且** payment_method=现金 | |
| ⚠️ 无发票 | bill_id ∈ {0, 260000000} | |
| 🔴 未付款 | outstanding_amount > 0 | |

## 每行一个物料明细格式

**原则**：一张发票有多行物料时，每行分开写，合计对上发票总额。

### 输出字段（14列）

| 字段名 | 说明 | 来源 |
|---|---|---|
| 凭证号 | 系统内部编号 | `name` |
| 发票号码 | 税务局认证编号 | `bill_no` |
| 日期 | 凭证日期 | `posting_date` |
| 供应商 | 供应商名称 | `supplier_name` |
| 发票总额 | 整张发票价税合计 | `grand_total` |
| 付款方式 | 电汇/现金/未付款/已确认已付 | PE 推算 |
| 付款凭证号 | 关联的 Payment Entry 编号 | PE.references |
| 物料名称 | 行项目物料 | `items[].item_name` |
| 规格型号 | 物料规格 | `items[].custom_guige_xinghao` |
| 备注 | 费用用途/说明 | `items[].custom_备注` |
| 明细摘要 | 备用补充说明 | `items[].custom_mingxi` |
| 数量 | 行数量 | `items[].qty` |
| 单价 | 行单价 | `items[].rate` |
| 金额 | 行金额（不含税） | `items[].amount` |

## ERPNext 字段区分（硬规则）

| 字段 | 含义 | 示例 |
|---|---|---|
| `name` | 凭证号（系统内部编号） | ACC-PINV-2026-00028 |
| `bill_no` | 发票号码（税务局认证） | 26322000000670322851 |
| `grand_total` | 发票价税合计（必须与 PDF 一致） | 101,882.70 |
| `outstanding_amount` | 未结算金额 | 0（已付）/ >0（未付）|
| `custom_备注` | 行项目备注（费用用途） | 3号浸渗机浸渗使用 |
| `custom_guige_xinghao` | 规格型号 | 200KG/桶 |

## payment_map 构建规范

1. 查询全量 Payment Entry（company=目标公司，不限日期）
2. 逐条 fetch PE 详情，获取 `mode_of_payment` 和 `references[]`
3. 建立 `invoice_name → {payment_method, pe_name}` 映射
4. **电汇优先**：若同一发票已有现金映射，遇到电汇 PE 时覆盖

---

## 注意事项

- 付款凭证日期可能晚于发票日期（银行转账滞后正常现象，不能因此判断为"未付款"）
- `outstanding_amount=0` 不代表已付——必须以 PE 实际关联的 `payment_method` 为准
- 无发票的明细（bill_no=0/260000000）需人工逐张确认是否可以补录发票
